Как работают препроцессоры?

Variables

			@my_blue: #4e6cb0; //LESS
			$my_blue: #4e6cb0; //SASS
			 my_blue: #4e6cb0; //Stylus
			h1 {
				color:@my_blue;
			}

Variables не только для цветовых переменных

			@fonts: helvetica, Arial, sans-serif;
			@images_path: "../img";
			@font-size: 1.5em;
			@margin: 20px;

Операции и цветовые функции

			width: 25px * 4 + 100px / 2 - 50px; //= 100px
			color: #990033 + #666666; //= #FF6699
			@light_blue: lighten(@my_blue, 20%);
			@fancy_blue: saturnate(@my_blue, 20%);

Mixins

Функции возвращают значение, миксины - CSS 
			.border-radius(@radius) {
			  -webkit-border-radius: @radius;
			     -moz-border-radius: @radius;
			          border-radius: @radius;
			}
			#header { .border-radius(4px); }

Наследование

			.pop {
				.pop-btn {
				}
			}
			//OUTPUT:
			.pop {}
			.pop .pop-btn {}

Наследование

			.link {
				&:hover {
				}
			}
			//OUTPUT:
			.link {}
			.link:hover {}

Недопонимание

Нам будет нужен Ruby или Node на бэкэнде

  • Sass написан на Ruby
  • LESS написан на Javascript
  • Всё это НЕ ЗАСТАВЛЯЕТ нас ставить Ruby или Node на прод
  • Это необходимо нам только на дев машине

LESS работает на клиенте

  • LESS может запускаться client-side
  • Это OK для тестинга или если мы не можем залезть на дев машину
  • НИКОГДА не используйте на проде. Это даже хуже. чем @import
  • Используйте Node. Или Rhino

Препроцессоры сложно установить

  • И Ruby и Node легко поставить на все платформы
  • Mac: Ruby из коробки, Node.pkg
  • Windows: RubyInstaller.exe, Node.exe
  • Unix: apt-get install ruby/nodejs

Как работают постпроцессоры?

Autoprefixer

  • autoprefixer
  • Плагин для PostCSS
  • Всё что он делает это:
  • Добавляет и убирает вендорные префиксы

Как это работает?

  • Вы конфигурируете браузерную матрицу под плагин
  • К примеру 2 последние версии или все > 5%
  • Плагин парсит наш CSS и смотрит в CanIUse
  • И после этого вносит изменения в наш CSS

Autoprefixer

			a {
				-webkit-border-radius: 5px;
						border-radius: 5px;
			}

			a { border-radius: 5px; }

Autoprefixer

			a { transform: scale(0.5); }

			a {
				-webkit-transform: scale(0.6);
				-moz-transform: scale(0.6);
				transform: scale(0.5);
			}
